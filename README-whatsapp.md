# WhatsApp OTP — Meta setup checklist

The code side is done and env-gated: until the variables below exist,
login keeps working exactly as today (demo echo). Complete these steps
at your own pace; nothing breaks while they are pending.

## One-time Meta setup (business.facebook.com)
1. **Business portfolio** — create one for Vakilim if you don't have it.
2. **Add WhatsApp** — Business settings > Accounts > WhatsApp accounts >
   Add. This creates your WhatsApp Business Account (WABA).
3. **Phone number** — WhatsApp > API Setup > Add phone number. Use a
   NEW number that has never been on consumer WhatsApp (a fresh SIM is
   easiest); you must receive its verification call/SMS once.
4. **Business verification** — Security Centre > Start verification.
   Upload company documents; approval typically takes 1–5 days. Until
   verified you can only message a short allow-list of test numbers —
   enough to test with your own phone.
5. **Authentication template** — WhatsApp Manager > Message templates >
   Create: category **Authentication**, name **otp_login**. Add the
   template in **three languages: Azerbaijani, Russian, English** (the
   body text is auto-generated by Meta for auth templates). Enable the
   **copy-code button**. Auth templates are usually approved within
   minutes.
6. **Permanent token** — Business settings > Users > System users >
   Add (role: Admin) > Assign assets: your WhatsApp account (full
   control) > Generate token with scopes
   `whatsapp_business_messaging` and `whatsapp_business_management`,
   expiry **Never**.
7. **Collect two values** — the token from step 6, and the
   **Phone number ID** shown in WhatsApp > API Setup.

## Wiring it up
Add locally to `.env` AND on Vercel (then Redeploy):

    WHATSAPP_TOKEN=EAAG...
    WHATSAPP_PHONE_NUMBER_ID=1234567890
    WHATSAPP_TEMPLATE=otp_login

Test: open /login on your phone's language, request a code — it should
arrive as a WhatsApp message with a copy button, in that language.

## Go-live flip
Once real delivery works on Vercel, set `OTP_DEV_ECHO=false` there and
Redeploy — the on-screen demo code disappears and WhatsApp becomes the
only channel. Locally you can keep the echo forever.

## Notes
- If Meta approved your English template as `en_US`, set
  `WHATSAPP_LANG_EN=en_US` (same pattern for az/ru if needed).
- If sending fails with a template-component error, your template may
  not have the button — delete the button block in
  `lib/otp-transport.ts` (marked) or recreate the template with the
  copy-code button enabled.
- Pricing: authentication conversations are billed per delivery at
  Meta's Azerbaijan rate — small, but real once volume grows.
- A number that is NOT on WhatsApp receives nothing; the UI still says
  the code was sent. Accepted trade-off per product decision.
